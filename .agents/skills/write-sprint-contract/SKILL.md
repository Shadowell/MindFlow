# Write Sprint Contract

## Purpose

Turn a vague request into a sprint contract that is small enough to implement and concrete enough to verify.

## Instructions

1. Read `docs/spec.md` and `docs/progress.md`.
2. Identify one focused outcome, not a full roadmap.
3. Write or update a file in `docs/contracts/`.
4. Include:
   - goal
   - in scope
   - out of scope
   - done means
   - verification
   - handoff
5. Keep the sprint small enough to finish in one iteration.

## Output

- the contract file path
- a short summary of the scope
- any missing assumptions that should be reviewed
